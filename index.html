JSON Web Token (JWT)

Terminology

Identification VS Authorization VS Authentication
  • Identification is a process that defines who wants to use source.

  • Authentication (AuthN) is a process that verifies that someone or something is who they say they are.

  • Authorization is the security process that determines a user or service's level of access.
    • Encode VS Encrypt

  • Encoding method - transforming data from one form to another. The main aim of encoding is to transform data into a form that is readable by most of the systems.
  • Encryption is an encoding technique in which a message is encoded by using an encryption algorithm in such a way that only authorized personnel can access the message or information

    • Hashing
  • In Hashing, data is converted to the hash using some hashing function, which can be any number generated from a string or text. Various hashing algorithms are MD5 and SHA256.
    • Token
  • An authentication token allows internet users to access applications, services, websites, and application programming interfaces (APIs) without having to enter their login credentials each time they visit.

    • Cookie sessions

    Session cookies are cookies that last for a session. A session starts when you launch a website or web app and ends when you leave the website or close your browser window. Session cookies contain information that is stored in a temporary memory location which is deleted after the session ends
    The most common example of a session cookie in action is the shopping cart on eCommerce websites. When you visit an online shop and add items to your shopping cart, the session cookie remembers your selection so your shopping cart will have the items you selected when you are ready to checkout. Without session cookies, the checkout page will not remember your selection and your shopping cart will be empty.
